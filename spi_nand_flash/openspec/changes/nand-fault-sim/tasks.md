## 1. Fault Simulator C Module

- [ ] 1.1 Create `host_test/main/nand_fault_sim.h` ‚Äî define `nand_fault_sim_config_t` with all fault fields: factory bad block list + count, `max_erase_cycles`, `max_prog_cycles`, `grave_page_threshold`; per-op failure probabilities (`read_fail_prob`, `prog_fail_prob`, `erase_fail_prob`, `copy_fail_prob`) and shared `op_fail_seed`; power-loss fields (`crash_after_ops_min`, `crash_after_ops_max`, `crash_probability`, `crash_seed`); ECC disturb thresholds (`ecc_mid_threshold`, `ecc_high_threshold`, `ecc_fail_threshold`); declare the eight `nand_impl` interface functions (`nand_is_bad`, `nand_mark_bad`, `nand_erase_block`, `nand_prog`, `nand_is_free`, `nand_read`, `nand_copy`, `nand_get_ecc_status`) ‚Äî these replace `nand_impl_linux.c` at link time and are the single interception point for all FTLs; declare stats API (`nand_fault_sim_get_erase_count`, `nand_fault_sim_get_prog_count`, `nand_fault_sim_get_read_count`); declare lifecycle API (`nand_fault_sim_init`, `nand_fault_sim_deinit`, `nand_fault_sim_reset`); declare preset API (`nand_fault_sim_config_preset`) and `nand_sim_scenario_t` enum
- [ ] 1.2 Create `host_test/main/nand_fault_sim.c` ‚Äî implement `nand_fault_sim_init`: call `nand_emul_init(handle, cfg->emul_conf)` to open the mmap backing, allocate per-block arrays (`erase_count[]`), per-page arrays (`prog_count[]`, `read_count[]`), initialize `crashed` flag and crash PRNG state from `crash_seed`, initialize op-failure PRNG state from `op_fail_seed`, resolve crash point for range mode. The file also provides the eight `nand_impl` symbol definitions (`nand_is_bad`, `nand_mark_bad`, `nand_erase_block`, `nand_prog`, `nand_is_free`, `nand_read`, `nand_copy`, `nand_get_ecc_status`) that replace `nand_impl_linux.c` at link time.
- [ ] 1.3 Implement `nand_is_bad` ‚Äî check factory bad-block list first (from config); otherwise delegate to `nand_emul_read` to inspect OOB marker bytes at `block * block_size + page_size` (same logic as `nand_impl_linux.c`); return true if marker ‚âÝ 0xFFFF
- [ ] 1.4 Implement `nand_mark_bad` ‚Äî implement identically to `nand_impl_linux.c`: erase the block via `nand_emul_erase_block`, then write a zero OOB marker at `block * block_size + page_size` via `nand_emul_write`; bad-block state is thus persisted in the mmap backing and survives remount with `keep_dump = true`
- [ ] 1.5 Implement `erase_block` ‚Äî check `crashed` flag ‚Üí `ESP_ERR_INVALID_STATE`; check op count / probability for crash trigger; if crash fires, clear seeded-random prefix of pages within block then return error; roll `op_fail_seed` PRNG against `erase_fail_prob` and return error without touching emulator if it fires; otherwise increment `erase_count[block]`, return error if count >= `max_erase_cycles`, else delegate to emulator
- [ ] 1.6 Implement `prog` ‚Äî check `crashed` flag ‚Üí `ESP_ERR_INVALID_STATE`; check op count / probability for crash trigger; if crash fires, write seeded-random byte prefix then return error; roll `op_fail_seed` PRNG against `prog_fail_prob` and return error without touching emulator if it fires; otherwise increment `prog_count[page]`, return error if count >= `max_prog_cycles`, else delegate to emulator
- [ ] 1.7 Implement `is_free` ‚Äî check `crashed` flag ‚Üí `ESP_ERR_INVALID_STATE`; delegate to underlying emulator
- [ ] 1.8 Implement `read` ‚Äî check `crashed` flag ‚Üí `ESP_ERR_INVALID_STATE`; roll `op_fail_seed` PRNG against `read_fail_prob` and return error without touching emulator if it fires; delegate to emulator; increment `read_count[page]`; if `prog_count[page] > grave_page_threshold` and `on_page_read_ecc != NULL`, invoke `on_page_read_ecc(page, NAND_ECC_NOT_CORRECTED, ctx)` (data bytes unchanged); otherwise invoke ECC disturb callback based on `read_count[page]` vs thresholds (see task 1.14)
- [ ] 1.9 Implement `nand_fault_sim_deinit` ‚Äî free all allocated arrays and deinit the underlying mmap emulator
- [ ] 1.10 Implement `nand_fault_sim_reset` ‚Äî zero `erase_count`, `prog_count`, `read_count`, `crashed` flag, and op counter; re-seed op-failure PRNG from `op_fail_seed`; re-derive crash point from config; preserve the mmap emulator state entirely (flash contents and OOB bad-block markers unchanged ‚Äî allows remount testing and preserves bad-block history across resets)
- [ ] 1.11 Implement power-loss crash trigger in `erase_block` and `prog` ‚Äî shared helper `ns_check_crash(sim)`: increment op counter; in range mode, compare to seeded crash point; in probability mode, roll `rand_r()` and compare to `crash_probability`; set `crashed = true` when triggered
- [ ] 1.12 Implement torn-write behavior in `prog` ‚Äî when crash fires, compute `torn_offset = 1 + rand_r(&crash_seed_state) % (len - 1)`; write bytes `[0, torn_offset)` to emulator via a partial-length call; return `ESP_ERR_FLASH_OP_FAIL`
- [ ] 1.13 Implement torn-erase behavior in `erase_block` ‚Äî when crash fires, compute `torn_pages = rand_r(&crash_seed_state) % pages_per_block`; call emulator erase only for pages `[0, torn_pages)` within block; return `ESP_ERR_FLASH_OP_FAIL`
- [ ] 1.14 Implement ECC read-disturb in `read` (called from task 1.8 when no grave-page condition) ‚Äî compare `read_count[page]` against `ecc_fail_threshold` first (highest priority), then `ecc_high_threshold`, then `ecc_mid_threshold`; invoke `on_page_read_ecc(page, status, ctx)` with the matching `nand_ecc_status_t` if non-NULL; if no threshold is crossed invoke with `NAND_ECC_OK` (or skip if callback NULL)
- [ ] 1.15 Implement `nand_fault_sim_config_preset(scenario)` ‚Äî return fully populated `nand_fault_sim_config_t` by value for each `nand_sim_scenario_t`: `NAND_SIM_SCENARIO_FRESH` (no faults), `NAND_SIM_SCENARIO_LIGHTLY_USED`, `NAND_SIM_SCENARIO_AGED`, `NAND_SIM_SCENARIO_FAILING`, `NAND_SIM_SCENARIO_POWER_LOSS`
